## 🎮 Реализованные функции

### 1. Команды краша

#### `!nuke` — Базовый краш
**Алгоритм:**
1. Проверка прав доступа (Freelist+)
2. Проверка кастомного текста (Whitelist+)
3. Удаление всех каналов параллельно
4. Удаление всех ролей параллельно
5. Создание 30 новых каналов
6. Спам 500 сообщений (распределённо по каналам)
7. Создание роли ☠️ Kanero и выдача запустившему
8. Логирование в MongoDB

**Производительность:**
- Время выполнения: ~15-30 секунд
- Параллельное выполнение: ✅
- Обработка ошибок: ✅

#### `!super_nuke` — Усиленный краш (Premium)
**Дополнительно:**
- Бан участников (оставляет 15 человек)
- Переименование сервера
- Приоритетное удаление

**Защищённые от бана:**
- Овнер бота
- Запустивший краш
- Владелец сервера
- Боты

#### `!owner_nuke` — Полный краш (Owner Nuke List)
**Особенности:**
- Банит ВСЕХ участников (кроме ботов и овнера сервера)
- Максимальная деструктивность
- Только для доверенных пользователей

### 2. Система подписок

#### Freelist (бесплатно)
- Получение: канал `#addbot`
- Автоматическая выдача роли **👥 User**
- Доступ к базовым командам
- Хранение в MongoDB

#### Whitelist (платно)
- Добавление: `!wl_add @user`
- Автоматическая выдача роли **✅ White**
- Кастомный текст спама
- Все команды Freelist

#### Premium (платно)
- Добавление: `!pm_add @user`
- Автоматическая выдача ролей **💎 Premium** + **✅ White**
- Удаление роли **👥 User** (апгрейд)
- Расширенные команды краша
- Массовые операции

#### Синхронизация ролей
**`!sync_roles`** — массовая синхронизация всех участников:
- Проверяет всех участников сервера
- Выдает роли согласно спискам
- Убирает роли у тех, кого нет в списках
- Удаляет из списков тех, кто покинул сервер

**`!fix_role [@пользователь]`** — проверка конкретного пользователя:
- Проверяет, есть ли пользователь в списках
- Выдает соответствующую роль, если её нет
- Убирает лишние роли
- Показывает статус в списках и текущие роли
- Работает для себя, если пользователь не указан

```python
# Иерархия прав
def is_premium(user_id):
    temp = check_temp_subscription(user_id)
    if temp == "pm":
        return True
    return user_id in PREMIUM_LIST or user_id == config.OWNER_ID

# Premium включает Whitelist
def is_whitelisted(user_id):
    temp = check_temp_subscription(user_id)
    if temp in ("wl", "pm"):
        return True
    return user_id in config.WHITELIST or user_id in PREMIUM_LIST
```

### Функции работы с БД:

```python
# Получение данных
async def db_get(collection: str, key: str, default=None):
    db = get_db()
    doc = await db[collection].find_one({"_id": key})
    return doc["value"] if doc else default

# Сохранение данных
async def db_set(collection: str, key: str, value):
    db = get_db()
    await db[collection].update_one(
        {"_id": key},
        {"$set": {"value": value}},
        upsert=True
    )
```

### Автоматическое сохранение:
```python
def save_whitelist():
    asyncio.create_task(db_set("data", "whitelist", config.WHITELIST))

def save_premium():
    asyncio.create_task(db_set("data", "premium", PREMIUM_LIST))

def save_temp_subscriptions():
    data = {
        uid: {"type": sub["type"], "expires": sub["expires"].isoformat()}
        for uid, sub in TEMP_SUBSCRIPTIONS.items()
    }
    asyncio.create_task(db_set("data", "temp_subscriptions", data))
```

## ⚠️ Известные проблемы

### 1. Ограничения Discord API
**Проблема:** Rate limits при массовых операциях  
**Решение:** Использование `asyncio.gather()` с `return_exceptions=True`  
**Статус:** ✅ Решено

### 2. Временные подписки не обновляются в реальном времени
**Проблема:** Проверка только при вызове команды  
**Решение:** Фоновая задача для проверки каждые 5 минут  
**Статус:** ⏳ Планируется

### 3. Кнопки компенсации не сохраняются при перезапуске
**Проблема:** View теряется при перезапуске бота  
**Решение:** Использование `custom_id` и восстановление View в `on_ready`  
**Статус:** ⏳ Планируется

### 4. Большой размер main.py
**Проблема:** 4450+ строк в одном файле  
**Решение:** Разделение на модули (cogs)  
**Статус:** ⏳ Планируется
