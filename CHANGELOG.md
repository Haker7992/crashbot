# 📝 История изменений — Kanero Bot

## [2.4.0] - 27.04.2026

### ✨ Добавлено
- `guild.chunk()` перед нюком — бот сразу получает всех участников сервера для максимальной скорости

### 🔧 Изменено

#### Текст спама (SPAM_TEXT)
- Полностью переработан — новый стильный дизайн с рамками и разделителями
- Добавлено упоминание **Интерактив клана** — краш от их имени
- Ссылки корректно отделены от текста (кликабельны в Discord)
- Убрана перезапись `SPAM_TEXT` из MongoDB при старте — текст всегда берётся из `config.py`

#### Параллельность нюков
- `!super_nuke` — всё строго параллельно: баны + удаление каналов + удаление ролей + создание каналов + спам одновременно
- Добавлена повторная попытка удаления ролей после основного gather
- `auto_nuke` и `auto_super_nuke` при входе бота на сервер теперь работают максимально быстро — `guild.chunk()` вызывается сразу, без задержек

### 🗑️ Удалено
- Команда `!owner_nuke`
- Команда `!auto_owner_nuke`
- Список `OWNER_NUKE_LIST`
- `AUTO_OWNER_NUKE` блок из `on_guild_join`
- Убран `asyncio.sleep(0.5)` из dm_all — больше не замедляет авто-нюк
- Убраны упоминания owner_nuke из `!auto_off` и `!auto_info`

### 🐛 Исправлено
- Исправлена проблема со старым текстом после перезапуска (MongoDB перезаписывал `config.py`)
- Исправлено неполное удаление ролей при параллельном выполнении
- **Исправлено отображение временных подписок в `!list`** — теперь временные подписки показываются только в отдельной секции "⏳ Временные подписки", а не дублируются в основных списках (Freelist/Whitelist/Premium)

## [1.0.0] - 26.04.2026

### ✨ Добавлено

#### Система временных подписок
- Добавлена структура `TEMP_SUBSCRIPTIONS` для хранения временных подписок
- Функция `add_temp_subscription()` для создания временных подписок
- Функция `check_temp_subscription()` для проверки и автоудаления истёкших
- Функция `save_temp_subscriptions()` для сохранения в MongoDB
- Функция `load_temp_subscriptions()` для загрузки из MongoDB
- Интеграция с `is_premium()`, `is_whitelisted()`, `is_freelisted()`

#### Команда компенсации
- Команда `!компенсация <тип> <время>` для выдачи временных подписок
- View с кнопкой для получения компенсации
- Автоматическая отправка логов в `#выдача-вайта`
- Защита от повторного получения компенсации
- Поддержка времени: 1h, 6h, 12h, 1d, 7d, 30d
- Поддержка типов: pm (Premium), wl (Whitelist), fl (Freelist)

#### Защита домашнего сервера
- Глобальная проверка прав на домашнем сервере (ID: 1497100825628115108)
- Блокировка деструктивных команд для не-овнеров
- Предупреждение при попытке использовать краш-команды
- Разрешение команд управления для владельца сервера
- Добавлено правило №7: "⛔ Попытка краша этого сервера запрещена"

#### Новые каналы
- Создан канал `🎁・компенсация` в категории ОСНОВНОЕ
- Настроены права доступа (только Owner может писать)

#### Команды управления для владельца сервера
- `!wl_add`, `!wl_remove`, `!wl_list` — управление whitelist
- `!pm_add`, `!pm_remove` — управление premium
- `!fl_add`, `!fl_remove`, `!fl_clear` — управление freelist
- `!auto_off`, `!auto_info` — управление авто-нюками
- `!list` — просмотр всех списков
- `!sync_roles` — синхронизация ролей

### 🔧 Изменено

#### Система прав
- Обновлена функция `is_whitelisted()` — теперь проверяет временные подписки
- Обновлена функция `is_premium()` — теперь проверяет временные подписки
- Обновлена функция `is_freelisted()` — теперь проверяет временные подписки

#### Глобальная проверка
- Переработана функция `global_guild_block()` для домашнего сервера
- Добавлен список `MANAGEMENT_COMMANDS` для команд управления
- Добавлен список `DESTRUCTIVE` для деструктивных команд
- Улучшена логика проверки прав

#### Логирование компенсаций
- Логи теперь отправляются только в `#выдача-вайта` (не в #sell)
- Добавлена информация о типе и сроке подписки

#### Импорты
- Добавлен импорт `from datetime import datetime, timedelta`

### 🐛 Исправлено

#### Проблемы с правами
- Исправлена проблема с командами управления на домашнем сервере
- Команды `wl_add`, `pm_add`, `fl_add` теперь работают для владельца сервера
- Убран декоратор `@wl_check()` с команд управления

#### Канал #addbot
- Исправлена работа канала #addbot на домашнем сервере
- Сообщения теперь корректно удаляются
- Freelist выдаётся автоматически

#### Защита от краша
- Деструктивные команды теперь блокируются на домашнем сервере
- Добавлено предупреждение вместо мута
